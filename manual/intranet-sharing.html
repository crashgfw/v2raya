v2rayA 局域网共享

发表于2020-11-16|更新于2024-11-08|用户手册
|阅读量:

介绍

无论是 安卓、苹果、平板、Windows、MacOS、Linux 还是 路由器，都是各自的软件可用，但是遇到一些特殊情况：苹果 iOS 下载不了软件、Switch 等设备不支持代理软件等，可以借助 v2rayA 的局域网代理功能解决这个问题。

这是一个临时解决方法，如果会使用路由器，强烈建议在路由器上安装 v2rayA 并开启局域网共享。

场景

有一台设备 A 和一台设备 B，设备 A 装了 v2rayA，现在要通过设备 A 实现设备 B 上网。

开启端口分享

确保设备 A 可以浏览器打开谷歌。然后开启 v2rayA 的 端口分享。

获取局域网 IP

这里获取的是设备 A 的局域网 IP，也就是安装了 v2rayA 的设备。

Windows

使用组合键 Win + R，Win 键先按住不放，再按 R 键。在搜索框输入 cmd 回车，在接下来的窗口里输入 ipconfig 找到 无线局域网适配器 WLAN 项，IPv4 地址 即为设备 A 在局域网里的 ip 地址。

MacOS

桌面右上角  —> 系统偏好设置…，选择网络，可以立即看到当前 Wi—Fi 下的局域网 IP 地址。局域网 IP 地址一般都以 192.168 开头。

设置设备 B

确保设备 A 和设备 B 接入相同的 Wi—Fi 下 。不同系统操作方法不太一样，一般的操作步骤：

  • 打开设置
  • 打开无线局域网
  • 点击已经连接的 Wi-Fi
  • 进入 Wi—Fi 设置页
  • 点击配置 HTTP 代理

HTTP 代理改 自动 为 手动，在服务器 / 主机 栏填写 设备 A 的局域网 IP，端口 栏填写 20171。如果有 认证，请关闭。最后保存。

不出意外，你的设备可以打开谷歌了。

Windows，Macos, Android, IOS 设备都可以使用端口分享通过 v2rayA 代理上网，只是 http，socks5 代理设置方式各不相同。
http/socks5 代理设置可以自行了解，只需将代理服务设置为设备 A 的局域网 IP，http 代理端口 设置为 20171 或者 20172（带分流规则）。socks5 代理端口 设置为 20170
